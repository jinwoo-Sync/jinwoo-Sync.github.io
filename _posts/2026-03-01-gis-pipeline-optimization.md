인도네시아 프로젝트에서 항공 측량 데이터를 처리하는 GIS 파이프라인을 구축하고, Python → C# 컨버팅을 통해 처리 속도를 33배 개선한 과정을 정리합니다.

## 배경

인도네시아 항공 측량 데이터는 LAS(포인트클라우드), TIF(래스터), SHP(벡터) 등 다양한 GIS 포맷으로 구성되어 있습니다. 연구팀에서 Python으로 개별 처리 스크립트를 개발했지만, 분할 개발된 코드들이 산재해 있었고 처리 성능에 한계가 있었습니다.

## 파이프라인 구조

전체 데이터 처리 시퀀스를 3단계로 정리했습니다:

### 1단계: SHP 파일 생성

```
TIF → PNG 변환 → Airborne + Global Building + OSM 결합 → .shp
```

TIF 이미지를 PNG로 변환한 뒤, 항공 데이터와 글로벌 건물 데이터, OpenStreetMap 데이터를 결합하여 SHP(Shapefile)를 생성합니다.

이 과정에서 겹치는 영역이 발생하는데, 중복 영역을 자동으로 합치는 기능을 구현하여 수작업을 제거했습니다.

### 2단계: LAS 핸들링

```
포인트클라우드 추출 → Polygon 분할 + Multi-layer 분할 → .las
```

SHP 결과물이 나오면 2단계와 3단계를 동시에 진행할 수 있습니다.

### 3단계: OBJ 메쉬 생성

```
LOD1 Mesh / LOD2 Mesh → .obj
```

## DTM 생성 최적화

DTM(Digital Terrain Model) 생성이 전체 파이프라인에서 가장 시간이 많이 소요되는 부분이었습니다.

### 기존 방식: PDAL CSF

기존에는 PDAL 명령어를 이용한 CSF(Cloth Simulation Filter) 방식으로 DTM을 생성했습니다. 이 방식은 69GB 데이터 기준 약 36시간이 소요되었습니다.

### 개선된 DTM 생성 알고리즘

C#으로 컨버팅하면서 DTM 생성 알고리즘을 2단계로 재설계했습니다:

**Step 1 — 건물 옥상 제거**

Voxel과 Block 단위의 통계 분석을 수행합니다. 포인트클라우드를 3D Voxel로 나누고 각 Voxel의 높이 분포를 분석하여 건물 옥상과 같은 '높은 평면'을 식별하고 제거합니다.

**Step 2 — 지면 점 격자화 및 보간**

추출된 지면 점을 격자(Grid)로 변환합니다. 이때 수역(강, 논)의 특징을 보존하면서 빈 공간을 보간하는 것이 핵심입니다.

## 성능 결과

| 지표 | 기존 (PDAL CSF) | 개선 후 (C# 파이프라인) |
|------|-----------------|----------------------|
| 입력 데이터 | 69 GB (원본) | 321 MB (전처리 후) |
| 처리 시간 | 2,160분 (36시간) | 3분 |
| 처리 속도 | 32.71 MB/min | 107.00 MB/min |
| DTM 생성 시간 | 9분 | 3분 |

**전체 파이프라인 기준 약 33배 성능 개선**을 달성했습니다.

## Python → C# 컨버팅에서 배운 점

1. **시퀀스 정리가 먼저**: 분할 개발된 코드의 동작 순서를 먼저 정리하고 통일한 뒤 컨버팅을 진행해야 합니다
2. **Python 가상환경 임베딩**: C# 프로그램에서 Python 스크립트를 직접 호출하는 구조를 만들면, 일부 라이브러리(PDAL, GDAL 등)는 Python 네이티브를 유지하면서도 전체 시스템은 C#으로 통합할 수 있습니다
3. **GIS 데이터의 특성**: LAS, TIF, SHP 등 GIS 데이터는 포맷별 특성을 이해해야 효율적인 파이프라인을 설계할 수 있습니다

## 기술 스택

- Python, C#, PDAL, GDAL/OGR
- SAM (Segment Anything Model), EfficientNet
- LAS, TIF, SHP, OBJ, PNG
